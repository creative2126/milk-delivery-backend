Razorpay India Configuration Guide

Important: This guide helps configure Razorpay for Indian payments. The "International cards not supported" error occurs when using non-Indian test cards or incorrect configuration.

Step-by-Step Configuration

1. Verify Account Settings

  • Log in to Razorpay Dashboard
  • Go to Settings → Account Settings
  • Ensure your account is registered in India
  • Verify your business is registered as Indian entity

2. Enable Payment Methods

  • Navigate to Settings → Payment Methods
  • Enable "Cards" under Payment Methods
  • Ensure "Domestic Cards" is enabled
  • Keep "International Cards" disabled for now (causes the error)

3. Use Correct Test Key

  • Go to Settings → API Keys
  • Copy the "Test Key ID"
  • Update in app.js if different

4. Use Indian Test Cards Only

These cards work with Indian Razorpay accounts:

  • Card: 4111 1111 1111 1111
  • Expiry: 12/25
  • CVV: 123
  • Name: Any Indian name

5. Test Payment Flow

  1. Open payment-debug.html
  2. Click "Test Payment"
  3. Use Indian test card details
  4. Complete payment

Alternative: Skip Payment for Testing

If payment configuration is complex, you can test the subscription flow without actual payment:

  • Comment out Razorpay code in app.js
  • Add direct success handler
  • Test all other features (location, building details, etc.)

Quick Fix

To bypass payment issues for testing:

  1. Use only Indian test cards
  2. Ensure Razorpay account is India-based
  3. Test with payment-debug.html first
Note: The "International cards not supported" error specifically indicates you're using non-Indian test cards with an Indian Razorpay account. Always use Indian test cards for testing.
